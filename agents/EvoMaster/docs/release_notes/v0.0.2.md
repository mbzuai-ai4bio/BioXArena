# EvoMaster v0.0.2 Release Notes and Quick Migration Guide

---

## I. Architecture Changes

### 1. Config System Overhaul: From Global Single LLM to Per-Agent Config

The old architecture had a single `agent` bound to one global LLM. The new version uses `agents` (plural), and each agent declares its own LLM, Tools, and Skills.

| Dimension | v0.0.1 (Old) | v0.0.2 (New) |
|-----------|--------------|--------------|
| Config field | `agent: {...}` | `agents: { name: {...} }` |
| LLM binding | Global `llm.default` | Each agent may specify `llm: "openai"` |
| Tools toggle | `enable_tools: true/false` | `tools: { builtin: [...], mcp: "..." }` |
| Skills | Global `skills.enabled` | Each agent may specify `skills: ["rag"]` or `"*"` |

Key file changes:
- `evomaster/config.py`: Added `ToolConfig`, removed `SkillConfig/KnowledgeSkillConfig/OperatorSkillConfig`; added per-agent helpers: `get_agent_config(name)`, `get_agents_config()`, `get_agent_llm_config(name)`, `get_agent_tools_config(name)`, `get_agent_skills_config(name)`.
- `EvoMasterConfig.agent` → `EvoMasterConfig.agents` (field rename)
- `EvoMasterConfig.skill` → `EvoMasterConfig.tools` (`ToolConfig` replaces `SkillConfig`)

### 2. Playground Base Class Overhaul: Multi-Agent Slot System

- Added `AgentSlots` container class with `dict`-like and attribute access (`self.agents.planning_agent`).
- `self.agent` (singular) → `self.agents` (`AgentSlots`), with backward compat: `self.agent = self.agents.get_random_agent()`.
- Base class `setup()` flow: `_setup_session()` → `_setup_agents()` → `(_setup_exps())`.
- New `_setup_agents()`: iterates agent config and creates instances, registering them in `self.agents`.
- New `copy_agent()`: deep-copies an agent (separate LLM and context, shared session/tools) for parallel runs.

### 3. Tool Registry Overhaul: Name-Based Filtering

- New `create_registry(builtin_names, skill_registry)` replaces `create_default_registry()`.
- Supports `builtin_names=["execute_bash", "finish"]` to register specific builtin tools.
- Agent gains `enabled_tool_names` to limit tools exposed to the LLM, decoupled from tools callable in code.
- All tools stay in the registry (callable from code); `enabled_tool_names` only filters LLM-visible tools.

### 4. Skills Unification

- **Removed** `KnowledgeSkill` and `OperatorSkill` distinction.
- **Unified** into `Skill` (former `OperatorSkill` renamed).
- `SkillMetaInfo` no longer has `skill_type`.
- `SkillRegistry` supports `skills` for name-based filtering and adds `create_subset()`.

---

## II. New Features

### 1. Parallel Experiment Execution

- `evomaster/env/local.py`: New `ResourceAllocator` for GPU/CPU allocation by parallel index.
- `LocalSession`: Thread-local storage (`_thread_local`) for `set_parallel_index()`, `set_workspace_path()`, etc.
- `LocalEnv`: New `setup_exp_workspace()` for `split_workspace_for_exp` (separate work dir per experiment).
- `LocalSessionConfig`: New `parallel` config (`enabled`, `max_parallel`, `split_workspace_for_exp`).
- `BasePlayground`: New `execute_parallel_tasks()` (parallel run via `ThreadPoolExecutor`).

### 2. Multimodal Support (Image Input)

- `evomaster/utils/llm.py`: New `encode_image_to_base64()`, `get_image_media_type()`, `build_multimodal_content()`.
- `evomaster/utils/types.py`: `BaseMessage.content` extended to `str | list[dict] | None` for multimodal blocks.
- `TaskInstance`: New `images: list[str]` field.
- `AnthropicLLM`: New `_convert_content_for_anthropic()` to convert OpenAI-style multimodal content to Anthropic format.
- `ContextManager` / `SimpleTokenCounter`: token estimation for images (~750 tokens).
- `run.py`: New `--images` CLI argument.

### 3. ML-Master Playground

- New `playground/ml_master/`: ML competition workflow (e.g. Kaggle).
  - Three-phase exps: `DraftExp` → `DebugExp` → `ImproveExp`.
  - UCT search manager (`core/utils/uct.py`): experiment search via MCTS.
  - Data preview tools (`core/utils/data_preview.py`): dataset summaries.
  - Visualization (`vis/`): tree-structured Web UI.

### 4. Multi-Agent Parallel Playground Example

- New `playground/minimal_multi_agent_parallel/`: demo using `copy_agent()` + `ThreadPoolExecutor` for parallel experiments.

## III. Important Changes

### Config File Format Changes

- `enable_tools: true/false` → `tools: { builtin: ["*"], mcp: "" }`
- Global `mcp:` config → per-agent `tools.mcp` (global `mcp:` kept as advanced option)
- `agent:` → `agents:` (field name in config)

### X-Master Playground

- `_setup_agents()` uses new `_create_agent()` signature (`llm_config` instead of `llm_config_dict`).
- Exps no longer pre-created in `setup()`; created at runtime via `_create_exp()` + `copy_agent()`.
- Parallel execution supported (previous TODOs removed).

### Dependency Changes

- Added `.env.template`.

---

## IV. Removed / Deprecated

| Removed | Description |
|---------|-------------|
| `KnowledgeSkill` | Unified into `Skill` |
| `OperatorSkill` | Unified into `Skill` |
| `SkillConfig` / `KnowledgeSkillConfig` / `OperatorSkillConfig` | Replaced by per-agent skills config |
| `ConfigManager.get_skill_config()` | No longer used |
| `BasePlayground._setup_llm_config()` | Replaced by `_setup_agent_llm(name)` |
| `_create_agent()` params `enable_tools` and `llm_config_dict` | Replaced by `tool_config` / `llm_config` / `skill_config` |
| `enable_tools: true/false` config field | Replaced by `tools: { builtin: [...] }` |

# Quick Migration Guide: v0.0.1 → v0.0.2

EvoMaster v0.0.2 improves code and config to better support customization and reuse while deprecating some APIs. This guide helps migrate to the new architecture.

## Step 1: Migrate Config (config.yaml)

### 1.1 `agent` → `agents`, Declare LLM per Agent

**Old (v0.0.1):**
```yaml
llm:
  openai:
    provider: "openai"
    model: "gpt-4"
    api_key: "${OPENAI_API_KEY}"
  default: "openai"

agents:
  planning:
    llm: "openai"
    max_turns: 10
    enable_tools: false     # <-- old style
    context: ...
  coding:
    llm: "openai"
    max_turns: 50
    enable_tools: true      # <-- old style
    context: ...
```

**New (v0.0.2):**
```yaml
llm:
  openai:
    provider: "openai"
    model: "gpt-4"
    api_key: "${OPENAI_API_KEY}"
  default: "openai"

agents:
  planning:
    llm: "openai"           # per-agent LLM
    max_turns: 10
    tools:                   # <-- new style: explicit control
      builtin: []            # no tools
    context: ...
  coding:
    llm: "openai"
    max_turns: 50
    tools:
      builtin: ["*"]         # all builtin tools
    context: ...
```

### 1.2 Tool Config: `enable_tools` → `tools`

| Old | New | Notes |
|-----|-----|-------|
| `enable_tools: true` | `tools: { builtin: ["*"] }` or `tools: "default"` | All builtin tools |
| `enable_tools: false` | `tools: { builtin: [] }` | No tools |
| N/A | `tools: { builtin: ["execute_bash", "finish"], mcp:"mcp_config.json" }` | Specific tools + MCP |
| Global `mcp:` | `tools: { mcp: "mcp_config.json" }` | Per-agent MCP |
| No `tools` key | Default `builtin: ["*"], mcp: ""` | All builtin |

### 1.3 Skills Config

| Old | New | Notes |
|-----|-----|-------|
| Global `skills: { enabled: true, skills_root: "..." }` | Per-agent `skills: ["rag"]` | Per-agent skill declaration |
| `skills.enabled: true` + shared registry | Per-agent `skills:` | Different agents can load different skill subsets |
| N/A | `skills: "*"` or `skills: ["*"]` | Load all skills (equiv. to old `enabled: true`) |
| N/A | `skills: ["rag", "pdf"]` | Load only specified skills |
| N/A | No `skills` key / `skills:` (empty) | Agent loads no skills |

**Old style:** Global config + manual loading in Playground
```yaml
# config.yaml
skills:
  enabled: true
  skills_root: "./evomaster/skills"
```
```python
# playground.py - manual loading
config_dict = self.config.model_dump()
skills_config = config_dict.get("skills", {})
skill_registry = None
if skills_config.get("enabled", False):
    skills_root = Path(skills_config.get("skills_root", "evomaster/skills"))
    skill_registry = SkillRegistry(skills_root)           # load all, shared by agents

self._setup_tools(skill_registry)                         # pass to tool registry

# Also pass when creating each agent
self._create_agent(..., skill_registry=skill_registry)
```

**New style:** Per-agent config, base class handles it
```yaml
# config.yaml
agents:
  search:
    llm: "openai"
    skills:            # agent loads all skills
      - "*"
  summarize:
    llm: "openai"
    skills:            # agent loads only rag and pdf
      - "rag"
      - "pdf"
  plan:
    llm: "openai"
    # no skills config → no skills
```
```python
# playground.py — no manual skill handling
def setup(self):
    self._setup_session()
    self._setup_agents()   # base class loads skill_registry per agent
```

## Step 2: Migrate Playground Code

### 2.1 Agent Storage: From Own Attributes to AgentSlots

**Old style:**
```python
class MyPlayground(BasePlayground):
    def __init__(self, ...):
        super().__init__(...)
        self.planning_agent = None
        self.coding_agent = None
        self.mcp_manager = None

    def setup(self):
        llm_config_dict = self._setup_llm_config()
        self._setup_session()

        # manual skill loading
        skill_registry = None
        config_dict = self.config.model_dump()
        skills_config = config_dict.get("skills", {})
        if skills_config.get("enabled", False):
            skills_root = Path(skills_config.get("skills_root", "evomaster/skills"))
            skill_registry = SkillRegistry(skills_root)

        # manual tool setup
        self._setup_tools(skill_registry)

        # manual agent creation from config
        agents_config = getattr(self.config, 'agents', {})
        if 'planning' in agents_config:
            planning_config = agents_config['planning']
            self.planning_agent = self._create_agent(
                name="planning",
                agent_config=planning_config,
                enable_tools=planning_config.get('enable_tools', False),
                llm_config_dict=llm_config_dict,
                skill_registry=skill_registry,
            )
        if 'coding' in agents_config:
            coding_config = agents_config['coding']
            self.coding_agent = self._create_agent(
                name="coding",
                agent_config=coding_config,
                enable_tools=coding_config.get('enable_tools', True),
                llm_config_dict=llm_config_dict,
                skill_registry=skill_registry,
            )
```

**New style:**
```python
class MyPlayground(BasePlayground):
    def __init__(self, ...):
        super().__init__(...)
        # 1. Declare agent slots (IDE-friendly)
        self.agents.declare("planning_agent", "coding_agent")
        self.mcp_manager = None

    def setup(self):
        # 2. Two lines — base class handles LLM/Tools/Skills
        self._setup_session()
        self._setup_agents()
        # Base class creates agents from config and registers in self.agents
        # Naming: config "planning" → self.agents.planning_agent
```

### 2.2 `_create_agent()` Parameter Changes

**Old signature:**
```python
self._create_agent(
    name="solver",
    agent_config=solver_config,
    enable_tools=solver_config.get('enable_tools', False),
    llm_config_dict=llm_config_dict,
    skill_registry=skill_registry,
)
```

**New signature:**
```python
self._create_agent(
    name="solver",
    agent_config=solver_config,     # optional, auto-loaded from config if omitted
    llm_config=llm_config,          # optional
    tool_config=tool_config,        # optional
    skill_config=skill_config,      # optional
)
```

Using `_setup_agents()` removes the need for manual `_create_agent()` calls.

### 2.3 Accessing Agents

**Old style:**
```python
# Direct attributes
self.planning_agent.run(task)
self.coding_agent.run(task)
```

**New style:**
```python
# Via AgentSlots (also supports attribute access)
self.agents.planning_agent.run(task)
self.agents.coding_agent.run(task)
```

## Step 3: Migrate Skill References

**Old style:**
```python
from evomaster.skills import KnowledgeSkill, OperatorSkill

# Type checks
if isinstance(skill, OperatorSkill):
    ...
elif isinstance(skill, KnowledgeSkill):
    ...
```

**New style:**
```python
from evomaster.skills import Skill

# Unified type, no distinction
if isinstance(skill, Skill):
    ...
```
