The competition task is a binary single-class classification problem where participants must predict whether a social commentary comment is insulting (label 1) or neutral (label 0), with predictions required as a real-valued probability between 0 and 1. The dataset provides a label column and two attribute fields: a timestamp in the format YYYYMMDDhhmmssZ (which may be blank) and the unicode-escaped comment text enclosed in double quotes. Submissions must be a single-column file of predictions in the range [0,1], formatted according to the provided sample. Performance is evaluated using the Area Under the Receiver Operating Curve (AUC), which penalizes high-confidence misclassifications, and the final ranking is determined by AUC on a withheld verification set released after model lock-in, requiring participants to submit self-contained code generating predictions on that set. Important guidelines define an insult as language intended to offend another conversation participant, excluding insults directed at non-participants; profanity alone does not constitute an insult, and the insulting nature should be obvious. The training data has label noise estimated below 1%, and the problem is prone to overfitting, with the final evaluation based on an unpublished, broadly sampled dataset to ensure generalizability.
